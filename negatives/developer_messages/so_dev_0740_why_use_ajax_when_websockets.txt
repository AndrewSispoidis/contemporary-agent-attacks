Why use AJAX when WebSockets is available?. I've been using WebSockets for a while now, I have chosen to create an Agile project management tool for my final year project at University utilizing Node server and WebSockets. I found using WebSockets provided a 624% increase in the number of requests per second my application could process. However since starting the project I've read of security loopholes, and some browsers choosing to disable WebSockets by default.. This leads me to the question: Why use AJAX when WebSockets seems to do such a great job of lowering latency and resource overhead, is there anything that AJAX does better than WebSockets?
